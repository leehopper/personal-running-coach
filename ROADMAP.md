## Post-Redesign Sequence (placeholders — noted 2026-07-07, not yet planned)

Builder-requested sequencing for what follows the SPLIT / Alpine UI redesign cycle. **None of these are planned or decided yet** — each gets its own brainstorm/plan (and research where flagged) when its turn comes. Order is the contract:

1. **LLM provider decision point — stay on the first-party Anthropic SDK vs refactor to OpenRouter** (for access to more affordable models). **Deliberately not decided now**; revisit only after the redesign ships. If taken, the refactor rides the existing `ICoachingLlm` adapter seam, but it touches load-bearing Anthropic-specific behavior that needs research at decision time: constrained-decoding/structured-output support (the Pattern-B determinism story, DEC-085), the streaming error taxonomy (R-084 / DEC-073), prompt caching (`cache_control` on the system block), and the committed eval-cache/replay discipline (DEC-039 / DEC-074 — a provider swap re-records everything). Related: § Cost optimization (DEC-038 tiered routing) below — OpenRouter is one candidate mechanism for it.
2. **Final MVP live validation pass** — after the provider decision is made (and the refactor, if taken, has landed): one full funded-key end-to-end pass as the last gate before MVP-0 is declared solid for real users. The redesign cycle's own closing pass (split-ui cycle plan, Slice 7) still closes that cycle and discharges the F-LIVE-1/F-LIVE-2 done-gate; this later pass is the *final* MVP validation, and is non-negotiable if the LLM transport changed underneath the app.
3. **Deployment** — actually host the application so the MVP users (builder + family/friends) can use it. Needs genuine research + thinking at that time: hosting target, managed Postgres/Redis, secrets handling, TLS/domain, CI/CD-to-deploy (Kubernetes stays deferred to public beta per DEC-032; Trivy image scanning activates per § Quality tooling when images ship). Technically after "MVP-0 done," tracked here so it isn't lost.
