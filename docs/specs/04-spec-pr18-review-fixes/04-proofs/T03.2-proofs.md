## Summary

Two fixes applied to `YamlPromptStore.cs`:

1. **Race condition fix (review finding #2):** Replaced `TryGetValue` / `LoadAndCacheAsync` / `TryAdd` pattern with `ConcurrentDictionary.GetOrAdd` using `Lazy<Task<PromptTemplate>>`. This ensures concurrent callers for the same cache key share a single load operation rather than racing to load and discard duplicates. Renamed `LoadAndCacheAsync` to `LoadAsync` since caching is now handled by `GetOrAdd`.

2. **Cache key validation (review finding #7):** Added input validation to `BuildCacheKey` that rejects `id` or `version` containing the `::` separator, preventing cache key injection/collision. Throws `ArgumentException` with descriptive messages.
