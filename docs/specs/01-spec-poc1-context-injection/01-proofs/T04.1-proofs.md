## Summary

### What was built

**Prompt Variation YAML Files (4 new):**
- `context-injection-8k.yaml` - Reduced budget (~8K tokens), Layer 2 only, max 2 weeks, max 3 conversation turns
- `context-injection-12k.yaml` - Medium budget (~12K tokens), mixed summarization, 1 week L1 + 3 weeks L2
- `context-injection-profile-end.yaml` - Profile data placed at END (recency attention zone)
- `context-injection-profile-middle.yaml` - Profile data placed in MIDDLE (low attention zone)

**Experiment Infrastructure (9 source files):**
- `ExperimentConfig.cs` - Record describing a single experiment variation
- `ExperimentCategory.cs` - Enum for 4 experiment categories (TokenBudget, PositionalPlacement, SummarizationLevel, ConversationHistory)
- `ProfilePlacement.cs` - Enum for profile position (Start, Middle, End)
- `SummarizationMode.cs` - Enum for training history mode (Mixed, PerWorkoutOnly, WeeklySummaryOnly)
- `ExperimentVariations.cs` - Static class defining all 11 variations across 4 experiments
- `ExperimentContextAssembler.cs` - Parameterized assembler supporting different budgets, placements, and modes
- `ExperimentResult.cs` - Record capturing full experiment result metadata
- `ExperimentRunner.cs` - Runner supporting dry-run and live-run modes with JSON output
- `SampleConversations.cs` - Test data: 5-turn conversations for intermediate and beginner profiles
