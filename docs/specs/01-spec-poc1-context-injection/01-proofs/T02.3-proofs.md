## Summary

All requirements implemented and verified:

1. **ICoachingLlm interface** - Single async method signature as specified: `Task<string> GenerateAsync(string systemPrompt, string userMessage, CancellationToken ct)`

2. **ClaudeCoachingLlm implementation** - Uses official Anthropic NuGet SDK (v12.9.0) with:
   - Model ID parameterized via settings (not hardcoded in C#)
   - API key from IConfiguration/IOptions (mapped from user-secrets `Anthropic:ApiKey`)
   - Low temperature (0.3) for deterministic-ish output
   - Built-in retry with exponential backoff via SDK (MaxRetries=3, handles 429/5xx)
   - API key never logged
   - High-performance LoggerMessage source generators for structured logging
   - IDisposable for proper client cleanup
   - Internal constructor for test injection of mock client

3. **CoachingLlmSettings** - Strongly-typed settings record with IOptions pattern

4. **NuGet packages added** - Anthropic v12.9.0, YamlDotNet v16.3.0 in central package management

5. **20 unit tests** all passing - covering GenerateAsync behavior, parameter forwarding, text extraction, input validation, settings validation, cancellation propagation
