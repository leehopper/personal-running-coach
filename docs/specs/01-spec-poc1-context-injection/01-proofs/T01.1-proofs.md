## Files Created

### Training Module (`Modules/Training/Models/`)

1. `PaceRange.cs` - Min/max pace per kilometer range
2. `TrainingPaces.cs` - Training pace zones derived from VDOT
3. `RaceTime.cs` - Recorded race result for fitness estimation
4. `InjuryNote.cs` - Injury or physical issue record
5. `UserPreferences.cs` - Scheduling and training preferences
6. `UserProfile.cs` - Runner's complete profile
7. `RaceGoal.cs` - Target race definition
8. `FitnessEstimate.cs` - Current fitness assessment with VDOT
9. `GoalState.cs` - Current training goal with optional race target
10. `WorkoutSummary.cs` - Per-workout training history (Layer 1)
11. `WeekSummary.cs` - Weekly training summary (Layer 2)
12. `DecimalRange.cs` - Generic min/max decimal range for distances/volumes

### Coaching Module (`Modules/Coaching/Models/`)

1. `MacroPlan.cs` - High-level phased training plan
2. `PlanPhase.cs` - Single phase within a macro plan
3. `MesoWeek.cs` - Weekly training template
4. `DaySlot.cs` - Daily slot within a weekly template
5. `MicroWorkout.cs` - Specific workout with structure and coaching notes
6. `WorkoutSegment.cs` - Segment within a structured workout

## Design Decisions

- Used `sealed record` types for immutability and value equality
- Used `ImmutableArray<T>` for collection properties (value semantics in records)
- Used `TimeSpan` for paces and durations per spec
- Used `DateOnly` for dates per spec
- Used `DateTime` for audit timestamps (CreatedOn, ModifiedOn)
- Created `DecimalRange` for `PlanPhase.WeeklyDistanceRangeKm` since `PaceRange` uses `TimeSpan`
- All types follow file-scoped namespace convention
- No Dto suffix on these types (they are domain records, not request/response DTOs)
