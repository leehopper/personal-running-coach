## Test Methods

1. **Sarah_Beginner_GeneratesSafeLowVolumePlan** -- Verifies WeeklyTargetKm within 10% ceiling, no Interval/Tempo workouts, at least 2 rest days
2. **Lee_Intermediate_GeneratesPacesWithinVdotZones** -- Verifies easy pace within VDOT range (15% tolerance), no pace faster than repetition zone floor
3. **Maria_Goalless_MaintainsCurrentFitnessWithVariety** -- Verifies WeeklyTargetKm within +-10% of 55km, more than 1 distinct WorkoutType
4. **James_Injured_GeneratesConservativeRecoveryPlan** -- Verifies MacroPlan TotalWeeks >= 4, all workouts <= 20 min and Easy only, coaching narrative acknowledges injury
5. **Priya_Constrained_RespectsExactly4RunDays** -- Verifies exactly 4 Run day slots and 3 Rest/CrossTrain day slots

## Key Design Decisions

1. **Structured output via IChatClient**: Uses `ChatResponseFormat.ForJsonSchema` with `JsonSchemaHelper.GenerateSchema<T>()` through the M.E.AI caching layer, rather than raw Anthropic API calls
2. **Deserialization with snake_case**: Matches `ClaudeCoachingLlm.StructuredOutputSerializerOptions` for consistent JSON mapping
3. **PlanConstraintEvaluator reuse**: Leverages existing deterministic evaluator for constraint checks, adds profile-specific assertions on top
4. **James dual-call pattern**: Separate structured (plan) and unstructured (narrative) calls with distinct scenario names for different cache keys
5. **All tests tagged `[Trait("Category", "Eval")]`**: Excluded from normal CI; require live API key
6. **Graceful skip**: All tests return early when `IsApiKeyConfigured` is false (CI without secrets)
